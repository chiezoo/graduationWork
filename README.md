## サービス概要
どんなサービスなのかを３行で説明してください。

病院のサイトを作りたいです。
そしてその中に順番待ちの状況がわかる何人待っていて、あなたは何番目なのかがわかるアプリです。
病院で待つ際、あと何人待ち時間があり、自分の番が来るのか、とても気になります。


## このサービスへの思い・作りたい理由
このサービスの題材となるものに関してのエピソードがあれば詳しく教えてください。
このサービスを思いつくにあたって元となる思いがあれば詳しく教えてください。

実際に使われているので世の中には存在していますが、このアプリがどういう仕組みで動くのか明確に理解し、
自分で再現してみたくなりました。
病院で後何人なのかスタッフの方に聞いたり、聞かずとも気になっている人も少なくないことから
とても有用なものであると認識しています。



## ユーザー層について
決めたユーザー層についてどうしてその層を対象にしたのかそれぞれ理由を教えてください。

病院を訪れる患者。順番待ちシステムが入っていない病院が多く、知りたい人が多いと思ったため。

## サービスの利用イメージ
ユーザーがこのサービスをどのように利用できて、それによってどんな価値を得られるかを簡単に説明してください。

オンライン予約と通常の訪問予約があり、病院側で番号を発行する。
病院スタッフ側では、順番が進んだら次の人が診察中であるように変更し、
患者は発行された番号が進んでいることを確認出来る。

架空の病院のホームページを作成し、患者が見えるようにする。
